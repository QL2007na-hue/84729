3.1 runtime/event_bus.py —— EventBus 事件总线
----------------------------------------------------------------------
状态: [已实现]

功能:
  - Redis Streams 消息发布 (xadd)
  - 消费者组订阅模式 (xreadgroup + xack)
  - 自动创建 Consumer Group (mkstream)
  - 每条事件包含 task_id / agent / event / payload / timestamp

API:
  emit_event(task_id, agent, event, payload) -> 返回消息 ID (xadd)
  subscribe(callback, block_ms)        -> 阻塞循环读取事件
  stop()                               -> 停止订阅循环

已知 Bug:
  [BUG-B1] subscribe() 的 try/except 裸捕获所有 Exception (L85)，
           StopIteration 和 KeyboardInterrupt 也被吞掉。
           导致 PluginLoader._event_loop() 的停止机制完全失效（见交叉模块分析）。

  [BUG-B2] 无重连机制。Redis 短暂断开后，订阅循环仅 sleep(1) 重试，
          但 _running 仍为 True，永远不退出，也永远不会重新连接。
          日志仅丢失（无 logging）。

  [BUG-B3] emit_event() 中 payload=None 会被 json.dumps(None) 序列化为
          字符串 "null"（不是 {}），消费者需要特别处理。

缺失功能:
  - 无 close() / 资源清理方法（redis_client 连接永不关闭）
  - 无 Dead Letter Queue
  - 纯同步实现，api/server.py 异步场景直接绕过本模块

4.1 EventBus 使用严重分裂
----------------------------------------------------------------------
当前三个子系统各自拥有独立的 Redis 消费者：

  系统              消费者组名           连接方式        用途
  ─────────────────────────────────────────────────────────────────
  EventBus (同步)    ai-runtime-group      redis 同步        Agent 发送事件
  api/server.py      api-ws-group          redis.asyncio    WebSocket 推送
  plugin_loader.py   plugin-group          redis 同步(新实例) 插件监听

这三个消费者组从同一条 Redis Streams (ai-runtime:events) 读取，
但属于完全独立的子系统，没有任何统一的生命周期管理。

影响:
  - 三个系统各自 xack，消息消费状态不共享
  - 某个消费者组挂掉不影响其他，但可能导致消息重复消费
  - 无统一监控或错误恢复

建议:
  统一通过单例 EventBus (或 EventBusHub) 管理所有消费者，
  API Server 和 PluginLoader 不应各自创建 Redis 连接。

4.4 致命缺失：Runtime Engine（运行时编排引擎）
----------------------------------------------------------------------
这是整个项目最严重的问题。

当前流程：
  用户 → API (create_task) → TaskManager (status=pending) → ????

缺失的编排逻辑:
  1. 任务创建后，没有任何东西调用 planner.plan()
  2. 规划完成后，没有任何东西按依赖顺序调用 coder.execute(subtask_id)
  3. 编码完成后，没有任何东西调用 reviewer.review(subtask_id)
  4. 审查不通过时，没有任何东西触发 coder.execute() 重试

每个 Agent 都是可独立调用的类，但缺少一个 Runtime Engine 把它们串联起来。

缺失的状态机:
  pending → planner.plan() → running (子任务 pending)
  subtask.pending → coder.execute() → subtask.running → subtask.review
  reviewer.review() → approve → done
                    → reject  → retry → coder.execute()

----------------------------------------------------------------------

4.5 PluginLoader 停止机制完全失效
----------------------------------------------------------------------
调用链:
  PluginLoader.stop()
    → self._running = False
    → _event_loop() 中 handle() 检测 _running → raise StopIteration()
    → StopIteration 传播到 EventBus.subscribe()
    → EventBus.subscribe() 的 except Exception 吞掉 StopIteration
    → sleep(1) → 继续循环
    → 后台线程永不退出

同时 start() 使用 daemon 线程，进程退出时被强制终止，
Redis 连接未关闭。

  task.coding_failed    Coder → 编码失败
    payload: {error}

    payload: {workspace}

14.3 手动驱动 Agent (当前模式，Engine 到位后自动化)
───────────────────────────────────────────────────────────────────
  from runtime.event_bus import EventBus
  from runtime.task_manager import TaskManager
  from workspace.manager import WorkspaceManager
  from agents.planner import Planner
  from agents.coder import CoderAgent
  from agents.reviewer import Reviewer

  bus = EventBus()
  tm = TaskManager()
  wm = WorkspaceManager()

  # Step 1: Plan
  planner = Planner(bus, tm, api_key="sk-xxx")
  result = planner.plan("写一个 Python 贪吃蛇游戏")
  root_id = result["task_id"]

  # Step 2: Code (每个子任务)
  coder = CoderAgent(tm, bus, wm, api_key="sk-xxx")
  for subtask in result["subtasks"]:
      coder.execute(subtask["id"])

  # Step 3: Review
  reviewer = Reviewer(bus, tm, wm)
  for subtask in result["subtasks"]:
      review_result = reviewer.review(subtask["id"])
      print(f"{subtask['id']}: {'PASS' if review_result['approved'] else 'FAIL'}")

  class MyPlugin(BasePlugin):
      name = "my-plugin"
      version = "1.0"

          return ["task.review_rejected"]

      def on_event(self, task_id, agent, event, payload):
          # 你的逻辑
          self.ctx.emit(task_id, "plugin.my_event", {"hello": "world"})

  # 注册到 PluginLoader:
  from plugin_sdk import PluginLoader, PluginContext
